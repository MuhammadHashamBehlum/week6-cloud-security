## Overview

This repository contains the Week 6 Monday assignment completed as part of the Sohail Smart Solutions Summer Training Program 2026.

The focus of this task is to build a strong understanding of cloud security fundamentals by exploring the **Shared Responsibility Model**, identifying common cloud security threats, and creating a basic cloud threat model for a fictional organization migrating its services to **Amazon Web Services (AWS)**.

## Learning Objectives

This assignment focuses on the following cloud security concepts:

- Understand the Shared Responsibility Model.
- Differentiate between cloud provider and customer security responsibilities.
- Identify common cloud security threats and risks.
- Learn how cloud governance supports secure cloud adoption.
- Explore ISO/IEC 27017 and ISO/IEC 27018.
- Develop a basic cloud threat model for a cloud migration scenario.

## Cloud Threats Identified

The following cloud-specific risks were identified during the migration assessment:

1. Misconfigured cloud storage exposing sensitive information.
2. Weak Identity and Access Management (IAM) permissions.
3. Insecure APIs used by cloud-hosted applications.
4. Sensitive data exposure due to improper encryption.
5. Unauthorized access through compromised user credentials.
6. Cloud infrastructure availability and service disruption risks.
